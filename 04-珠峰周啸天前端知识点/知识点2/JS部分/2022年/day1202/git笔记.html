(把工作区所有修改的和新增的文件提交到暂存区，不包括删除的) git status(查看当前文件提交状态)【红色代表在工作区；绿色代表在暂存区； 没有文件，并显示clean之类的单词，说明文件已经提交到历史区或者没有任何更改】 clear:清屏 3、把文件提交到历史区 ***git commit -m '描述' (把文件提交到历史区，并生成此次提交的历史版本) git log(查看提交的历史版本信息，不包括回滚) git reflog(查看所有的历史版本信息,包括回滚信息) 4、回滚指定历史版本的代码 git reset --hard 七位历史版本号 5.与远程仓库关联 1.gitee 创建一个远程仓库 2. git remote -v(查看本地和远程仓库的连接状态) 3. git remote add origin 远程仓库地址(和远程仓库的某个新项目建立连接，origin这个名可以改，但是大家都统一叫这个) git remote add origin https://gitee.com/zfyk/a2021_06demoa.git 4. git push -u origin "master"/git push origin "master" (第一次【要输入用户名和密码 username password】) git push(以后) 推送本地的代码到远程仓库 5. git pull origin master (第一次，拉取远程仓库的代码到本地) 6.git clone git网址（第一回，文件夹里什么也没有） https://gitee.com/zfyk/a2021_06demoa.git git clone 远程仓库地址 仓库的名字(如果不写默认原仓库名) + 其真正的开发中，大家都使用这个 + 你们的项目老大把项目骨架搭建好之后你们每一个项目小组成员都去把远程的代码拉取到你们的本地去开发 + git clone 任何人，只要知道了仓库网址，就可以将资料克隆（复制，开源）下来， 如果在此基础上进行修改，是无法提交的（没有权限），需要对方给你开通权限 6. git分支 1. 查看分支 git branch 2. 创建分支 git branch dev(分支名) 3. 切换分支 git checkout dev(分支名)---->写内容，提交(git push origin dev) 3.1 创建分支，并进行切换到新分支： git checkout -b dev 4.
